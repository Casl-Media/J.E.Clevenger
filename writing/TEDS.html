Some of the ideas have started to come alive and are hosted on GitHub

A sample of which can be seen below. Given a standard structure a standard visualization should be available, the medium between the the structure and the visual is a data format. The format should contain the structure, file based structures seem to be self-explanatory. JSON is a good example for a file-based medium to contain tree data.

D3 provides a nice collapsible tree visualization. Mike Bostock (D3 creator) implements this using the flare.json data format.

I think the true power in visualizations is allowing others to implement the same method with their own data. Everyone should be leveraging these powerful visualizations for knowledge, sharing all along the way.

Casl Media created this script to help the world explore tree data structures. The current implementation uses a folder structure to create data in flare.json format.

    some far reaching ideas associated with this implementation:
  • help people explore their computers, operating systems, etc (you can use the script to explore any directory, e.g. your operating system base directory)
  • expand on nodes to include more meta data, why things are where they are in the tree
  • demonstrate how the flare.json format captures the tree structure

click on the nodes. folder structure can be found in the repo.
